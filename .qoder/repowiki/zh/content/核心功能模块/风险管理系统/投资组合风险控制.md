## 简介
本文件为《现代海龟协议》投资组合风险控制模块的技术文档，围绕五重熔断阀值设计（单一市场容量熔断、高关联市场熔断、中等关联市场熔断、弱关联市场熔断、单向总敞口熔断）与三层相关性矩阵算法展开，系统阐述风险单位概念、风险归一化处理方法、实时风险监控与熔断触发机制、自动拦截逻辑，以及投资组合分散化策略与宏观系统性风险防范措施。文档同时提供完整的代码实现路径与可视化图表说明，帮助读者在极端市场条件下理解风险控制的效果与边界。

## 详细组件分析

### 五重熔断阀值设计
- 单一市场容量熔断（4个风险单位）
  - 含义：同一标的累计风险单位不得超过阈值。
  - 实现：按 ticker 聚合当前活跃持仓，与阈值比较。
- 高度关联市场熔断（6个风险单位）
  - 含义：与目标资产高度相关（|相关系数| ≥ 0.7）的资产群组累计风险单位不得超过阈值。
  - 实现：查询相关性表，计算最高相关系数并选择对应阈值。
- **中等关联市场熔断（8个风险单位）** - 新增
  - 含义：与目标资产中等程度相关（0.4 ≤ |相关系数| < 0.7）的资产群组累计风险单位不得超过阈值。
  - 实现：新增的中间层级限制，提供更精细的风险控制。
- 弱关联市场熔断（10个风险单位）
  - 含义：对无明确相关性数据或低相关性资产，采用宽松阈值。
  - 实现：无相关性数据时使用该阈值。
- 单向总敞口熔断（12个风险单位）
  - 含义：任一方向（LONG 或 SHORT）的总风险单位不得超过阈值。
  - 实现：按方向聚合活跃持仓，分别与阈值比较。

### 三层相关性矩阵算法实现
- 皮尔逊相关系数计算
  - 输入：两资产的收益序列（长度需相同且≥10）。
  - 输出：相关系数与关联类型（HIGH/MEDIUM/LOW）。
- 实时相关性监控
  - 通过 MarketCorrelation 表存储与查询，支持按资产对检索。
  - **新增有效期过滤机制**：只返回未过期的相关性数据。
- 动态阈值调整机制
  - 根据最高相关系数绝对值选择阈值：
    - ≥0.7 使用高度关联阈值（6单位）
    - 0.4 ≤ |相关系数| < 0.7 使用中等关联阈值（8单位）
    - <0.4 使用弱关联阈值（10单位）

### 风险单位（Risk Unit）与归一化
- 风险单位定义
  - 以 N 值（ATR）与每点美元价值为基准的标准化头寸单位，确保跨资产风险可比。
- 归一化处理
  - 通过 N 值与合约乘数映射，将不同波动率资产的潜在损失统一换算为美元波动率，实现跨市场风险平价。
- 在风控中的应用
  - 活跃持仓以风险单位计量，熔断检查基于单位总数而非名义头寸规模。

## 性能考量
- 计算复杂度
  - 相关性计算：O(n)，其中 n 为收益序列长度；熔断检查：O(m)，m 为活跃持仓数量。
  - **新增中等关联限制**增加一次相关系数比较操作，但对整体性能影响微小。
- 数据访问
  - 通过索引优化的查询减少数据库负载；批量查询活跃持仓与相关性矩阵。
  - **新增有效期过滤**通过数据库索引优化，避免全表扫描。
- 并发与扩展
  - FastAPI 异步特性与 SQLAlchemy 会话管理支持高并发请求处理。

## 故障排查指南
- 常见问题
  - 相关性数据缺失：系统自动采用弱关联阈值，不影响下单流程。
  - 收益序列长度不足：相关性计算返回低关联类型，避免误判。
  - **新增中等关联限制问题**：检查相关性数据是否正确分类到中等关联层级。
  - **相关性数据过期**：确认 cleanup_db.py 脚本正常运行，相关性数据被及时清理。
  - 阈值告警：检查配置项与数据库相关性表的有效期与准确性。
- 排查步骤
  - 确认活跃持仓与相关性表数据是否正确。
  - 核对配置类中的阈值设置是否符合预期。
  - **新增中等关联限制检查**：验证相关性数据的阈值分类是否正确。
  - **相关性数据有效性检查**：确认 `valid_until` 字段是否正确设置。
  - 查看 API 返回的阻断原因与剩余可加仓单位数，定位具体限制。

- 单一标的过度集中
  - 单一市场容量熔断防止对某一标的的过度加仓，降低黑天鹅冲击的集中度风险。
